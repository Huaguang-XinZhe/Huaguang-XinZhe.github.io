    仪表盘

    当前核心：求职

    路径（大方向）：

    • 两端协同个人管理系统（框架）；

      • 时间：自动从移动端时光流获取每日时间记录数据，渲染展示并提供修改、存档、统计功能；
      • 财务：同上（时间）；
      • 待办：汇集数据，集中处理；
      • 对标、复查；

      ==亮点==：实现自动分类，并融入 AI 辅助数据操作。

    • 使用 自开发 & 开源 主题的个人博客；

      构建知识库，使用 AI 搜索、问答。

    • 多星且活跃的 GitHub 账号【开源】；

    当前时间统析的流程是怎样的？

    1. 从移动端获取昨日的时间数据；

    2. 创建一个视图，手动类属和操作（合并、拆分等）；

      这一步只要做少量的工作，维护的越久越是这样，大多数的工作（类属）在移动端就已经自动完成了。

    3. 所有的类属工作完成之后，点击统计，即将分组后的结果插入到 categroy_curation 表中，并可视化呈现；

    4. 构建灵活的架构，自动跟踪相应的类属时长……

    从移动端获取昨日的时间数据

    前端不直接和移动端打交道，而是和后端交互，调用后端的 API，获取昨日数据；

    这就要求在获取的时候，服务器本地数据库就已经存在要获取的数据；

    这些数据应该由移动端在合适的时机自动上传（就在晚睡结束之后吧，刚刚好）；

    移动端

    更新移动端的自动类属；

    实现插入、新增事件的自动判别；

    更新撤销方法，总结、分析，实现撤销可复用；

    让新的移动端可以上手使用；

    前端

    可视化呈现昨日事件列表；

    实现事件拆分、合并；

    实现类属完成后的统计（饼图）；
